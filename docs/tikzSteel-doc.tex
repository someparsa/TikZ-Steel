\section{Purpose and Scope}

TikZ-Steel provides TikZ commands for drawing structural cross-section
sketches. The current package covers cold-formed steel (CFS), hot-rolled steel
(HRS), and initial reinforced-concrete (RC) cross-sections.

The package has two command layers:

\begin{itemize}
  \item Public key-value commands such as \command{TikZSteelChannel}. These are
  recommended for new documents because the options are readable and can be
  omitted when defaults are acceptable.
  \item Legacy positional commands such as \command{csChannel}. These remain
  available for compatibility with older documents and lower-level drawings.
\end{itemize}

\section{Installation and Loading}

Place \texttt{tikzSteel.sty} next to your document or in a directory searched by
your LaTeX installation, then load it in the preamble:

\begin{lstlisting}
\usepackage{tikzSteel}
\end{lstlisting}

TikZ-Steel loads TikZ and the TikZ \keyname{calc} library.

\section{Quick Start}

The example below draws a lipped cold-formed channel with a label, centerlines,
and the current basic dimension overlay.

\begin{center}
\begin{adjustbox}{max width=45mm,max height=40mm}
\begin{tikzpicture}
  \TikZSteelLippedChannel[
    depth=245,
    flange=75,
    lip=20,
    thickness=2.5,
    radius=5,
    centerline=true,
    dimensions=true,
    label=C245,
    label x=35,
    label y=130,
    scale=0.03
  ]
\end{tikzpicture}
\end{adjustbox}
\end{center}

\begin{lstlisting}

\section{Key-Value Model}

Every public drawing command accepts a single optional key list:

\begin{lstlisting}
\TikZSteelChannel[depth=180, flange=55, thickness=2, radius=4]
\end{lstlisting}

The keys are reset to package defaults each time a public drawing command is
called. Use \keyname{x}, \keyname{y}, \keyname{scale}, and \keyname{rotate} to
position a section inside a larger \texttt{tikzpicture}.

\section{Style Hooks}

The package routes drawing through TikZ styles. This lets users keep the same
geometry while changing color, line style, or fill appearance.

\begin{longtable}{p{0.28\linewidth}p{0.60\linewidth}}
\toprule
Style & Purpose \\
\midrule
\keyname{tikzSteel/straight} & Straight steel segments \\
\keyname{tikzSteel/round} & Curved/radius steel segments \\
\keyname{tikzSteel/centerline} & Optional centerlines \\
\keyname{tikzSteel/hidden} & Hidden or reference lines \\
\keyname{tikzSteel/dimension} & Current dimension overlay lines \\
\keyname{tikzSteel/label} & Labels drawn with the \keyname{label} key \\
\keyname{tikzSteel/fill} & Fills for closed steel/bar shapes \\
\keyname{tikzSteel/concrete} & RC concrete outline and fill \\
\keyname{tikzSteel/rebar} & RC rebar circles \\
\keyname{tikzSteel/tie} & RC tie/stirrup outline \\
\bottomrule
\end{longtable}

\begin{lstlisting}
\TikZSteelSetup{
  tikzSteel/straight/.style={tikzSteel/default, black},
  tikzSteel/round/.style={tikzSteel/default, gray},
  tikzSteel/fill/.style={fill=gray!15}
}
\end{lstlisting}

\section{Dimension Overlay}

The \keyname{dimensions=true} key currently draws a generic horizontal and
vertical dimension guide around the section. It is not a complete engineering
dimensioning system. It does not yet draw dimension values, extension lines,
standard offsets, unit formatting, or standard-specific notation.

\section{Current Limitations}

\begin{itemize}
  \item The \keyname{dimensions=true} overlay is a placeholder, not a
  standard-based engineering dimensioning system.
  \item The \keyname{simplified} and \keyname{detailed} mode keys are accepted
  for API stability, but detailed rolled-radius behavior is still future work.
  \item Generated figures are schematic cross-section sketches. They are not
  section-property calculators and do not validate engineering dimensions.
\end{itemize}
